# 产品/精算辣评 三语改造 — 本次一次性完成内容

## 一、后台（API + 数据）

### 1. 产品接口支持语言参数

- **接口**：`GET /api/v1/products`、`GET /api/v1/products/{slug}`
- **新增参数**：`lang=cn`（简体）| `lang=hk`（繁体）| `lang=en`（英文），默认 `cn`
- **作用**：返回的 `tags`、`features`、`content` 按所选语言给出；`content` 从 `content_zh` / `content_tw` / `content_en` 里按 `lang` 选

### 2. 标签 / 精算辣评数据结构

- **标签（tags/features）** 支持两种存法：
  - 旧：`["管理权益户口", "含人民币"]`（纯中文数组）
  - 新：`[{"zh":"管理权益户口","en":"Managed Equity Account","hk":"管理權益戶口"}]`（三语对象数组）
- **精算辣评**：产品表已有 `content_zh`、`content_en`、`content_tw`，API 按 `lang` 返回对应字段到 `content`

### 3. AI 解析（新数据）

- **ai_parser**：prompt 要求输出 `content_zh` / `content_en` / `content_tw`，以及 `tags`、`features` 为新格式 `[{"zh","en","hk"}]`
- **ingest**：入库时若 AI 只给中文，会调 `translator.tags_to_i18n()` 翻成三语再存；同时会写 `content_en`、`content_tw`

### 4. 翻译服务

- **translator.py**：`translate_text()`、`tags_to_i18n()`，通过本机 Ollama 做中→英/繁，超时已设为 180 秒

### 5. 历史数据迁移（本次正在跑的）

- **脚本**：`scripts/migrate_i18n_tags_content.py`
- **做两件事**：
  1. **标签**：把旧格式标签转成新格式；若已是新格式但 en/hk 还是中文，则重新翻译修补
  2. **精算辣评**：有 `content_zh` 但缺 `content_en`/`content_tw` 的，用 Ollama 补齐
- **跑完后**：49 个产品的标签和精算辣评在库里都是三语齐全（或能补的都补了）

---

## 二、本机 / 环境（为迁移能跑通）

- **.env**：`MAC_IP=192.168.8.194`（你本机 IP，供 NAS 上的 API 连 Ollama 时用）
- **Ollama**：本机已设 `OLLAMA_HOST=0.0.0.0`（写进 `~/.zshrc`），监听 `*:11434`，方便 NAS 访问
- **迁移执行方式**：用一次性 Docker 容器 + `MAC_IP=host.docker.internal` 连本机 Ollama，写同一份数据库（避免 NAS 连不上 Mac 时跑不了）

---

## 三、后面增量怎么弄

- **新产品**：走 AI 解析 + ingest，会直接按三语格式入库，无需再跑迁移。
- **偶尔补翻**：若以后又有一批旧数据要补 en/hk，再跑一次 `migrate_i18n_tags_content.py` 即可（脚本会跳过已经合格的，只补缺的）。
- **怎么再跑迁移**：见 `scripts/README_MIGRATE_I18N.md`（先停 API，再跑一次性容器，再开 API）。

## 四、你这边怎么“看到”效果

- **前台网站**：产品列表 / 详情 切到 **英文** 或 **繁体**，标签和精算辣评会显示对应语言。
- **接口**：  
  `GET /api/v1/products?type=savings&lang=en`  
  `GET /api/v1/products/{slug}?lang=hk`  
  返回里的 `tags`、`content` 即为该语言。
